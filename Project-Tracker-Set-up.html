Elizabeth Greenwood

2 March 2020

1. Installing R

  1. Press the windows logo key on your keyboard
  2. Type Software
  3. Click on the Software Center
  4. Click on R (3.5.1) and R Studio (1.1.453)
  5. Click Install and wait for it to finish installing

After that, youre all done!

2. Installing the necessary packages

The following packages are required to run the project tracker:

  • shiny
  • tidyverse
  • shinythemes
  • plotly
  • DT
  • stringr
  • shinydashboard
  • treemap
  • lubridate

To install these packages, type the following into the console of R Studio:

install.Packages("[package_name]") (swapping [package_name] with the names listed above)

Alternatively, you can just copy and paste the following code into the console and press enter:

install.packages("shiny")
install.packages("tidyverse")
install.packages("shinythemes")
install.packages("plotly")
install.packages("DT")
install.packages("stringr")
install.packages("shinydashboard")
install.packages("treemap")
install.packages("lubridate")

3. Downloading the App.R file

Now that R is all set up you need to download the App.R file from the teams page, which can be found here.

To download the file hover over the App.R file in the directory and theree dots should appear. Click the three dots a drop down box should appear and download is one of the options so click that.

Once the file has been downloaded it save the file (without changing the name) in a place which is easy to navigate to.

4. Syncing the teams page to your library

The data used in the project tracker is stored in the SOAD general section in SharePoint so go to this link and click the sync button.

If you do not need all the files in the SOAD teams site. It would be better if you only sync the required folders. This means that syncing the documents will be much quicker and produce less errors. To do this

  1. Click the blue cloud on the right of your task bar.
  2. Click settings
  3. You should see something like this:

Click Choose folders under the location Strategic Operations Analysis Division - General and then unselect any of the files you do not need to sync to your library. If you do not need any files, you just need to ensure the Project Tracker folder is selected.

5. Editing the App.R file

This is the last step before you can run the app!

Open the App.R file that you downloaded and saved in section 3 in R Studio. Once you have done this all you need to do is enter your username within the speech marks on line 12.

Now click Run App at the top right of the editor section and you can now use the project tracker!

6. (Optional) Opening the project tracker without opening R Studio

To make it easier to use the project tracker if you follow the next steps you will be able to click a short cut on your desktop which automatically opens the application without opening R Studio and having to run the file:

  1. Create a run.R file and save in the local directory where App.R is saved. Click here for the example run.R file. The folder address is the file path to the folder where your App.R file is stored. (If you click on this folder there is a button in the ribbon of file explorer to copy the link). In the below examples the App.R file and the other files which you are about to create are stored in a folder in documents called Run Project Tracker`

    require(shiny)
folder_address = 'C://Users//[username]//OneDrive - Department for Education//Documents//Run Project Tracker'
runApp(folder_address, launch.browser=TRUE)

  2. Create a file named Project Tracker.bat with the following text:

    • A string containing the file-path to your R files. (the one below will most likely be very similar to your file-path, so if you are struggling to find it use that as guidance)
    • CMD BATCH
    • A second string containing the file path to the run.r file you created in step 1
    • An example can be seen below:
    "C:\Program Files\R\R-3.5.3\bin\R.exe" CMD BATCH "C:\Users\[username]\OneDrive - Department for Education\Documents\Run Project Tracker"
    • These files can be created using notepad and can be saved using the following file extensions: .R for run.R and .bat for Project Tracker.bat

  3. Then you can create a shortcut by right clicking where you want to add the shortcut then clicking New -> Shortcut -> Browse -> from this you can select your Project Tracker.bat file you have just created -> Next -> Finish.
